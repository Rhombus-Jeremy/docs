## Step 1: Get Your API Key

<Steps>
  <Step title="Access Rhombus Console">
    Navigate to [console.rhombussystems.com](https://console.rhombussystems.com) and log in with your credentials.
  </Step>
  
  <Step title="Generate API Key">
    1. Go to **Settings** â†’ **API Keys**
    2. Click **Generate New API Key**
    3. Give your key a descriptive name (e.g., "Development Integration")
    4. Copy and securely store your API key
    
    <Warning>
      **Important**: Your API key provides full access to your organization's data. Store it securely and never commit it to version control.
    </Warning>
  </Step>
</Steps>

## Step 2: Make Your First API Call

Test your setup with a simple request to get your organization details:

<CodeGroup>
```bash cURL
curl -X GET "https://api2.rhombussystems.com/api/org" \
  -H "x-auth-apikey: YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import requests

# Replace with your actual API key
API_KEY = "your_api_key_here"
BASE_URL = "https://api2.rhombussystems.com"

headers = {
    "x-auth-apikey": API_KEY,
    "Content-Type": "application/json"
}

response = requests.get(f"{BASE_URL}/api/org", headers=headers)

if response.status_code == 200:
    org_data = response.json()
    print(f"Organization: {org_data['name']}")
    print(f"Org ID: {org_data['uuid']}")
else:
    print(f"Error: {response.status_code} - {response.text}")
```

```javascript JavaScript
const API_KEY = 'your_api_key_here';
const BASE_URL = 'https://api2.rhombussystems.com';

async function getOrganization() {
  try {
    const response = await fetch(`${BASE_URL}/api/org`, {
      method: 'GET',
      headers: {
        'x-auth-apikey': API_KEY,
        'Content-Type': 'application/json'
      }
    });

    if (response.ok) {
      const orgData = await response.json();
      console.log(`Organization: ${orgData.name}`);
      console.log(`Org ID: ${orgData.uuid}`);
    } else {
      console.error(`Error: ${response.status} - ${await response.text()}`);
    }
  } catch (error) {
    console.error('Request failed:', error);
  }
}

getOrganization();
```

```go Go
package main

import (
    "encoding/json"
    "fmt"
    "io"
    "net/http"
)

const (
    APIKey  = "your_api_key_here"
    BaseURL = "https://api2.rhombussystems.com"
)

type Organization struct {
    Name string `json:"name"`
    UUID string `json:"uuid"`
}

func main() {
    client := &http.Client{}
    
    req, err := http.NewRequest("GET", BaseURL+"/api/org", nil)
    if err != nil {
        fmt.Printf("Error creating request: %v\n", err)
        return
    }
    
    req.Header.Set("x-auth-apikey", APIKey)
    req.Header.Set("Content-Type", "application/json")
    
    resp, err := client.Do(req)
    if err != nil {
        fmt.Printf("Error making request: %v\n", err)
        return
    }
    defer resp.Body.Close()
    
    body, err := io.ReadAll(resp.Body)
    if err != nil {
        fmt.Printf("Error reading response: %v\n", err)
        return
    }
    
    if resp.StatusCode == 200 {
        var org Organization
        if err := json.Unmarshal(body, &org); err != nil {
            fmt.Printf("Error parsing JSON: %v\n", err)
            return
        }
        fmt.Printf("Organization: %s\n", org.Name)
        fmt.Printf("Org ID: %s\n", org.UUID)
    } else {
        fmt.Printf("Error: %d - %s\n", resp.StatusCode, string(body))
    }
}
```

## Step 3: Explore Key Endpoints

Now that you're connected, try these essential endpoints:

<Tabs>
  <Tab title="Cameras">
    List all cameras in your organization:
    
    ```bash
    curl -X GET "https://api2.rhombussystems.com/api/camera/getMinimalCameraStateList" \
      -H "x-auth-apikey: YOUR_API_KEY"
    ```
    
    **Use cases**: Device management, live streaming, footage retrieval
  </Tab>
  
  <Tab title="Users">
    Get users in your organization:
    
    ```bash
    curl -X GET "https://api2.rhombussystems.com/api/user/getUsersInOrg" \
      -H "x-auth-apikey: YOUR_API_KEY"
    ```
    
    **Use cases**: User management, access control, permissions
  </Tab>
  
  <Tab title="Events">
    Search for recent events:
    
    ```bash
    curl -X POST "https://api2.rhombussystems.com/api/event/getPolicyAlertsV2" \
      -H "x-auth-apikey: YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"limit": 10}'
    ```
    
    **Use cases**: Security monitoring, alert management, analytics
  </Tab>
  
  <Tab title="Access Control">
    List access control groups:
    
    ```bash
    curl -X GET "https://api2.rhombussystems.com/api/accessControl/findAccessControlGroupsByOrg" \
      -H "x-auth-apikey: YOUR_API_KEY"
    ```
    
    **Use cases**: Door management, credential assignment, access logs
  </Tab>
</Tabs>

## Common Integration Patterns

<AccordionGroup>
  <Accordion icon="video" title="Live Video Streaming">
    **Goal**: Display live camera feeds in your application
    
    1. Get camera list with `getMinimalCameraStateList`
    2. Create shared stream with `createSharedLiveVideoStream`
    3. Use returned stream URL in your video player
    
    **Best for**: Security dashboards, monitoring applications
  </Accordion>
  
  <Accordion icon="search" title="Event-Driven Workflows">
    **Goal**: Respond to security events in real-time
    
    1. Set up webhooks for real-time notifications
    2. Use `getPolicyAlertsV2` to fetch event details
    3. Implement custom business logic based on event types
    
    **Best for**: Automated responses, notification systems
  </Accordion>
  
  <Accordion icon="door-open" title="Access Management">
    **Goal**: Integrate with existing HR/identity systems
    
    1. Sync users with `createUser` and `updateUser`
    2. Manage credentials with access control endpoints
    3. Monitor access events for compliance reporting
    
    **Best for**: Enterprise integrations, compliance systems
  </Accordion>
</AccordionGroup>
